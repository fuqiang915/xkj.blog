PhoneGap插件开发

最近项目有需求，要做一个Android版的PhoneGap聊天插件，简单来说就是封装一些Android原生页面.对于我这个没有接触过PhoneGap的人来说还是费了一番力气的。

最终做完这个之后几大感想:

做技术的只有Google能救你，baidu还是算了吧。我能说百度连PhoneGap官网都搜不到么？

PhoneGap文档做完我才知道不同版本的内容是相对互补的,而我只看了最新版本的文档,里面的东西寥寥无几…(被这个坑的很惨)

PhoneGap插件实际上就是帮你把原生代码导入到PhoneGap Android平台下的项目中一起编译运行。因此，library,sub-project,source code 都是可以直接放到插件里面的.

环境需求

  • phonggap基础开发环境: cordova
  • 插件创建工具: plugman

插件配置

官方plugin.xml配置详解

说几个重点的配置项,下面是典型的plugin.xml

当你的插件需要用到sub-project时,需要配置<framework src="src/android/libs/mylibrary_project" custom="true" />

当你的插件需要用到jar时,需要配置<source-file src="src/android/libs/xxxx.jar" target-dir="libs" />

sub-project 的配置是难住我的地方.

sub-project 的结构需要是ant而不是gradle,也就是eclipse下的Library Project结构
