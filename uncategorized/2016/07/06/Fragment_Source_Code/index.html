Fragment

片段，官方提供的容器用于展示页面。典型的宿主是FragmentActivity。与Activity的功能类似，但是比Activity更加灵活，可以自由实例化，动态添加、切换、组合等。生命周期随Activity和自己的状态改变。

实际上Fragment不止以Activity为宿主，只要实现了FragmentHostCallback就可以作为Fragment的宿主

原理

Android 3.0(API 11)以后引入，主要是为大屏幕（Pad）提供UI解决方案。

[B] 最好将Fragment设计成可重复使用的模块化Activity组件,一个Fragment可能添加到多个Activity，不要在Fragment直接关联另一个Fragment或Activity，而以接口的方式关联。

参考：官方文档

{: .pull-right}

生命周期

生命周期如右图,

一般必须实现的三个生命周期

onCreate

用于恢复一些，上次暂停或停止后保存的数据

onCreateView

创建Fragment的View,实现View的查找等

onPause

保存一些需要恢复的数据,确保用户回来之后需要存在的数据

扩展类

  • DialogFragment
    用于替代Dialog,优点在于可以将对话框压入Fragment管理栈

  • ListFragment
    显示一个列表，封装了列表的一些操作，比如实现了onListItemClick方法等

  • PreferenceFragment
    用于创建设置页面，PreferenceActivity的替代品

添加用户界面

inflater.inflate(int,ViewGroup,boolean)
三个参数

  • 布局
  • Fragment中的View的父View
  • boolean参数:指示是否应该在扩展期间将扩展布局附加至 ViewGroup（第二个参数）的布尔值。（在本例中，其值为 false，因为系统已经将扩展布局插入 container—传递 true 值会在最终布局中创建一个多余的视图组。）

添加到Activity中

###1,布局文件中标签形式添加

当系统创建此 Activity 布局时，会实例化在布局中指定的每个片段，并为每个片段调用 onCreateView() 方法，以检索每个片段的布局。系统会直接插入片段返回的 View 来替代 <fragment> 元素。

注：每个片段都需要一个唯一的标识符，重启 Activity 时，系统可以使用该标识符来恢复片段（您也可以使用该标识符来捕获片段以执行某些事务，如将其删除）。

可以通过三种方式为片段提供 ID：

  • 为 android:id 属性提供唯一 ID
  • 为 android:tag 属性提供唯一字符串
  • 如果您未给以上两个属性提供值，系统会使用容器视图的 ID

2,通过编程方式添加到ViewGroup

添加没有UI的Fragment

当Fragment没有UI时，只能通过add(Fragment fragment,String tag)的方式添加到Activity中，此时Tag就是唯一标识符。此时也不会调用onCreateView方法。可以通过findFragmentByTag()方式查找到此Fragment

FragmentActivity

  • 继承链:
    BaseFragmentActivityHoneycomb > BaseFragmentActivityEclair > BaseFragmentActivityDonut > Activity
  • 如果想作为Fragment的Host需要实现FragmentHostCallback,HostCallbacks是其中的实现类

实现Fragment的onActiivtyResult回调

  • int mNextCandidateRequestIndex
  • boolean mStartedActivityFromFragment
  • SparseArrayCompat<String> mPendingFragmentActivityResults
  • requestCode有效的是0xffff 低4位16进制,否则无效

基于Activity的onActivityResult根据requestCode从mPendingFragmentActivityResults中找到对应Fragment名称，进而找到Fragment

FragmentController

内部有属性FragmentHostCallback mHost, 用于包裹FragmentHostCallback的功能,让Activity可以作为Fragment的宿主。基本上实现的方法都是直接调用mHost,或mHost.mFragmentManager的方法。作为二者功能的一个整合。

FragmentManager,FragmentManagerImpl

  • abstract FragmentManager : 定义接口
  • interface BackStackEntry : 回退栈接口

  • class FragmentManagerImpl : 实现类

  • moveToState
    实现了Fragment的生命周期相关调用
