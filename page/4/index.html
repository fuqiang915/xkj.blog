Adb Shell 命令总结

目录

Shell

  • 登入shelladb shell //运行远端shell交互
  • 运行远端shell 命令: adb shell <command>
  • 运行仿真控制台命令: adb emu <command>

进程信息相关

  • 获取手机所有进程: adb shell ps -x
    • 利用grep筛选关键字：adb shell ps -x | grep qiyi
    • VSS - Virtual Set Size 虚拟耗用内存（包含共享库占用的内存）
    • RSS - Resident Set Size 实际使用物理内存（包含共享库占用的内存）
    • PSS - Proportional Set Size 实际使用的物理内存（比例分配共享库占用的内存）
    • USS - Unique Set Size 进程独自占用的物理内存（不包含共享库占用的内存）
  • 设备信息：cat /system/build.prop | grep "product"
  • CPU信息：/proc/cpuinfo和/proc/stat
  • 内存：/proc/meminfo adb shell dumpsys meminfo

文件导入导出

  • 导入: adb push <local> <remote>
    • adb push <local_file_path> /sdcard/

  • 导出: adb pull <remote> [<local>]

    • adb pull /sdcard/ <local_file_path>

  • 文件同步: adb sync [<directory>] //只有发生改变时从主机拷贝到设备

日志

  • 浏览日志: adb logcat [<filter-spec>]

应用卸载安装

  • 拷贝文件包到设备并安装: adb install [-l] [-r] [-s] <file>
  • 卸载程序包，-k意味着保留数据和缓存: adb uninstall [-k] <package>
  • 返回所有的bugreport信息: adb bugreport
  • 转发套接字连接: adb forward <local> <remote>

连接设备

  • 阻塞直到设备上线: adb wait-for-device
  • 启动服务： adb start-server
  • 关闭服务：adb kill-server
  • 列印offline,bootloader,device信息: adb get-state
  • 获取序列号：adb get-serialno
  • 连续列印设备状态:adb status-window
  • 重装载/system分区: adb remount
  • 重启: adb reboot [bootloader|recomry]
    • adb reboot-bootloader
  • 获取权限：adb root
  • 重启adb重新监听usb：adb usb

Android线程池解析

目录

概念

思路: 线程池维护N个线程持续运行，可以随时接收Runnable任务交给池内线程执行。任务队列为空时等待。

优点

  • 重用线程池中的线程，减少因为线程的创建和销毁带来的性能开销
  • 能有效控制线程池的最大并发数，避免大量的线程之间因互相抢占系统资源而导致的阻塞线程
  • 能够对线程进行简单地管理，并提供定时执行以及指定间隔循环执行等功能 源于Java Executor接口，Android中的实现：ThreadPoolExecutor。 可配置为四类线程池。

相比来说通过new Thread的方式执行的缺点：

  • 每次new Thread新建对象性能差。
  • 线程缺乏统一管理，可能无限制新建线程，相互之间竞争，及可能占用过多系统资源导致死机或oom。
  • 缺乏更多功能，如定时执行、定期执行、线程中断。

构造函数说明

  • corePoolSize
    核心线程数。默认核心线程一直存活，即便闲置时。若allowCoreThreadTimeOut=true => 核心线程等待新任务时间超过keepAliveTime会被终止。
  • maximumPoolSize
    最大线程数,活动线程达到此值后，后续新任务会被阻塞
  • keepAliveTime
    非核心线程闲置时超时时长，超过后非核心线程会被回收。当allowCoreThreadTimeOut = true 那么核心线程也会超时。
  • unit
    keepAliveTime的单位,枚举。TimeUnit.MILLISECONDS,TimeUnit.SECONDS,TimeUnit.MINUTES
  • workQueue
    线程池中的任务队列，通过线程池的execute方法提交的Runnable会存储在这个参数中
  • threadFactory
    线程工厂，为线程池提供创建新线程功能。ThreadFactory是接口,只有一个方法： Thread newThread(Runnable r)

Java线程池的种类

  • newCachedThreadPool

缓存的线程池，需要时创建，无用时回收，合理使用资源.

  • newFixedThreadPool

固定数量线程

  • newScheduledThreadPool

用于定时，延时，周期性任务的执行

  • newSingleThreadPool

只有一个线程运行，保证任务顺序执行FIFO，LILO

什么是 Cmd Markdown

您可以使用很多工具书写 Markdown，但是 Cmd Markdown 是这个星球上我们已知的、最好的 Markdown 工具——没有之一 ：）因为深信文字的力量，所以我们和你一样，对流畅书写，分享思想和知识，以及阅读体验有极致的追求，我们把对于这些诉求的回应整合在 Cmd Markdown，并且一次，两次，三次，乃至无数次地提升这个工具的体验，最终将它演化成一个 编辑/发布/阅读 Markdown 的在线平台——您可以在任何地方，任何系统/设备上管理这里的文字。

1. 实时同步预览

我们将 Cmd Markdown 的主界面一分为二，左边为编辑区，右边为预览区，在编辑区的操作会实时地渲染到预览区方便查看最终的版面效果，并且如果你在其中一个区拖动滚动条，我们有一个巧妙的算法把另一个区的滚动条同步到等价的位置，超酷！

2. 编辑工具栏

也许您还是一个 Markdown 语法的新手，在您完全熟悉它之前，我们在 编辑区 的顶部放置了一个如下图所示的工具栏，您可以使用鼠标在工具栏上调整格式，不过我们仍旧鼓励你使用键盘标记格式，提高书写的流畅度。

3. 编辑模式

完全心无旁骛的方式编辑文字：点击 编辑工具栏 最右侧的拉伸按钮或者按下 Ctrl + M，将 Cmd Markdown 切换到独立的编辑模式，这是一个极度简洁的写作环境，所有可能会引起分心的元素都已经被挪除，超清爽！

4. 实时的云端文稿

为了保障数据安全，Cmd Markdown 会将您每一次击键的内容保存至云端，同时在 编辑工具栏 的最右侧提示 已保存 的字样。无需担心浏览器崩溃，机器掉电或者地震，海啸——在编辑的过程中随时关闭浏览器或者机器，下一次回到 Cmd Markdown 的时候继续写作。

5. 离线模式

在网络环境不稳定的情况下记录文字一样很安全！在您写作的时候，如果电脑突然失去网络连接，Cmd Markdown 会智能切换至离线模式，将您后续键入的文字保存在本地，直到网络恢复再将他们传送至云端，即使在网络恢复前关闭浏览器或者电脑，一样没有问题，等到下次开启 Cmd Markdown 的时候，她会提醒您将离线保存的文字传送至云端。简而言之，我们尽最大的努力保障您文字的安全。

PhoneGap插件开发

最近项目有需求，要做一个Android版的PhoneGap聊天插件，简单来说就是封装一些Android原生页面.对于我这个没有接触过PhoneGap的人来说还是费了一番力气的。

最终做完这个之后几大感想:

做技术的只有Google能救你，baidu还是算了吧。我能说百度连PhoneGap官网都搜不到么？

PhoneGap文档做完我才知道不同版本的内容是相对互补的,而我只看了最新版本的文档,里面的东西寥寥无几…(被这个坑的很惨)

PhoneGap插件实际上就是帮你把原生代码导入到PhoneGap Android平台下的项目中一起编译运行。因此，library,sub-project,source code 都是可以直接放到插件里面的.

  • phonggap基础开发环境: cordova
  • 插件创建工具: plugman

官方plugin.xml配置详解

说几个重点的配置项,下面是典型的plugin.xml

当你的插件需要用到sub-project时,需要配置<framework src="src/android/libs/mylibrary_project" custom="true" />

当你的插件需要用到jar时,需要配置<source-file src="src/android/libs/xxxx.jar" target-dir="libs" />

sub-project 的配置是难住我的地方.

sub-project 的结构需要是ant而不是gradle,也就是eclipse下的Library Project结构
